2. Pipeline Audit de Contenu 🟡 Architecture validée

Rôle final : Évaluer les pages existantes d'un site sur tous les aspects SEO/AEO/GEO/EEAT/UX et produire un rapport d'audit complet avec recommandations actionnables. Le pipeline NE PUBLIE PAS — il diagnostique. La réécriture est gérée par le Pipeline 1 (Éditorial).

Connexion avec le Pipeline 1 : L'audit produit un audit_brief structuré. Le pipeline Éditorial le consomme pour réécrire les pages faibles.
