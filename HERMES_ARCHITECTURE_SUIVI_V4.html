7. 18 regles de robustesse (toutes implementees)

  1. ✅ Contrats Pydantic — tous les agents ont des modeles valides
  2. ✅ Superviseur — consentement obligatoire, bornes, rate limiting
  3. ✅ Defensif uniquement — pas de test d'intrusion
  4. ✅ Confidence scoring — High/Medium/Low sur chaque issue/alerte/correlation
  5. ✅ Source de preuve — agent source + URL + observed + rule
  6. ✅ do_not_recommend_if — pas de reco absurde
  7. ✅ Adaptatif — seuils par type de page
  8. ✅ CMS-aware — recos localisees (WP->Yoast, PrestaShop->Module SEO)
  9. ✅ Fallback cascade — GSC->TalorData->DataForSEO->openserp
  10. ✅ Auto-disable — KE apres 402, PSI apres 429
  11. ✅ Deduplication — T15 + S07
  12. ✅ Exports multiples — HTML, JSON, CSV, PDF
  13. ✅ Routage inter-pipelines — T18 + S10
  14. ✅ Rate limiting + robots.txt
  15. ✅ Skip intelligent — agents conditionnels, T20 logs, S03/S08 fast
  16. ✅ Tracabilite — chaque issue documentee
  17. ✅ Core Update suspension — S02b + S07 protegent contre les faux positifs massifs
  18. ✅ Encouragements — pas seulement des alertes, des insights positifs (paysage concurrentiel favorable)

10. Methode de travail

  1. L'utilisateur definit le process (quel pipeline, quels agents, quel ordre)
  2. L'utilisateur ecrit le cahier des charges
  3. Claude implemente agent par agent, dans l'ordre defini
  4. Claude fait les tests et verifications
  5. Claude connecte au pipeline existant sans casser ce qui marche
  6. L'utilisateur valide chaque agent ou groupe d'agents
  7. L'utilisateur teste en conditions reelles
