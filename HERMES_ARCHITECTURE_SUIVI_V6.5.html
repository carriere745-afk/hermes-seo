9. 24 regles de robustesse

  1. ✅ Contrats Pydantic — tous les agents et projets ont des modeles valides
  2. ✅ Superviseur — consentement, bornes, rate limiting (tous pipelines)
  3. ✅ Defensif uniquement — pas de test d'intrusion
  4. ✅ Confidence scoring — High/Medium/Low + 0-100 (P5, P6)
  5. ✅ Source de preuve — agent source + URL + observed + rule
  6. ✅ do_not_recommend_if — pas de reco absurde
  7. ✅ Adaptatif — seuils par type de page et profil
  8. ✅ CMS-aware — recos localisees (WP, PrestaShop...)
  9. ✅ Fallback cascade — GSC → TalorData → DataForSEO → mock
  10. ✅ Auto-disable — KE apres 402, PSI apres 429
  11. ✅ Deduplication — T15 + S07
  12. ✅ Exports multiples — HTML, JSON, CSV, PDF
  13. ✅ Routage inter-pipelines — T18 + S10 + ST11 + B11
  14. ✅ Rate limiting + robots.txt + AI crawlers (B16)
  15. ✅ Skip intelligent — agents conditionnels (fast/premium/compliance/V1.5/V2/V3)
  16. ✅ Tracabilite — chaque issue/alerte/recommandation documentee
  17. ✅ Core Update suspension — S02b + S07
  18. ✅ Encouragements — insights positifs
  19. ✅ Decision Trace — chaque recommandation P5 explicable
  20. ✅ Observability — hermes_events + predictions_history (P5) + campaign_results (P6)
  21. ✅ AI Link Intelligence — llms.txt, 11 AI crawlers, AI Visibility Score (B16)
  22. ✅ Follow/Nofollow — ratio + alerte desequilibre (B02 + B14)
  23. 🆕 Disclaimers — 8 types, tracabilite par projet, affichage contextuel
  24. 🆕 Project Model — scores consolides, etat pipelines, onboarding guide, flags

10. Methode de travail

  1. L'utilisateur definit le process (quel pipeline, quels agents, quel ordre)
  2. L'utilisateur ecrit le cahier des charges
  3. Claude implemente agent par agent, dans l'ordre defini
  4. Claude fait les tests et verifications
  5. Claude connecte au pipeline existant sans casser ce qui marche
  6. L'utilisateur valide chaque agent ou groupe d'agents
  7. L'utilisateur teste en conditions reelles
