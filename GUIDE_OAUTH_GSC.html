Hermes SEO — Guide OAuth GSC pas à pas

22 juin 2026 · Adapté à Hermes SEO v3 · Streamlit local + Streamlit Cloud

Étape 1 — Créer le projet Google Cloud

1. Va sur console.cloud.google.com
2. En haut à gauche, clique sur le sélecteur de projet → Nouveau projet
3. Nom du projet : hermes-seo (ou ce que tu veux)
4. Clique Créer. Attends 5 secondes que le projet soit actif.
5. Vérifie en haut que le projet hermes-seo est bien sélectionné.

Étape 2 — Activer les API Search Console

1. Dans le menu gauche → API et services → Bibliothèque
2. Cherche Google Search Console API → clique Activer
3. Cherche Google Analytics Data API → clique Activer
4. Cherche URL Inspection API → clique Activer

Résultat : 3 API activées pour le projet hermes-seo.

Étape 3 — Créer l'écran de consentement OAuth

1. Menu gauche → API et services → Écran de consentement OAuth
2. Type d'utilisateur : Externe (tu veux que tes clients puissent se connecter)
3. Remplis les champs obligatoires :
ChampValeur
Nom de l'applicationHermes SEO
Email d'assistanceTon email (celui du compte Google Cloud)
Logo (optionnel)Laisse vide
Domaines autorisésLaisse vide (pas nécessaire pour une app serveur)
Liens (page d'accueil, etc.)Tu peux les laisser vides
Email du développeurTon email
4. Clique Enregistrer et continuer
5. Écran "Portées" : clique Ajouter ou supprimer des champs d'application
6. Cherche et coche Search Console API (readonly suffit pour commencer)
7. Clique Enregistrer et continuer
8. Écran "Utilisateurs tests" : ajoute ton email → Enregistrer et continuer
Important : Tant que ton app n'est pas "vérifiée" par Google, seuls les utilisateurs tests (ton email) pourront se connecter. C'est suffisant pour le développement. Pour la prod multi-client, il faudra faire vérifier l'app (processus Google ~2-4 semaines).

Étape 4 — Créer l'identifiant OAuth 2.0

1. Menu gauche → API et services → Identifiants
2. Clique + Créer des identifiants → ID client OAuth
3. Type d'application : Application Web
4. Nom : Hermes SEO Web

Origines JavaScript autorisées

EnvironnementValeur à saisir

Streamlit Cloudhttps://carriere745-afk-hermes-seo-app-XXXX.streamlit.app (remplace XXXX par ton vrai ID d'app)
Ajoute les deux — tu peux mettre plusieurs origines. Clique sur "+ Ajouter une origine" pour chacune.

URI de redirection autorisés

Voici les valeurs exactes pour Hermes SEO :

EnvironnementURI à saisir
Local (Streamlit)http://localhost:8502/auth/google/callback
Streamlit Cloudhttps://carriere745-afk-hermes-seo-app-XXXX.streamlit.app/auth/google/callback
Critique : l'URI de redirection doit correspondre exactement à l'endpoint que tu vas créer dans Hermes. Si tu changes le chemin, change-le des deux côtés. Une différence d'un slash (/) peut bloquer l'auth.
5. Clique Créer
6. Une popup apparaît avec ton Client ID et ton Client Secret. Copie-les tout de suite.

Étape 5 — Récupérer les identifiants

Google t'affiche une popup avec :

3. Sur Streamlit Cloud : Manage App → Secrets → ajoute :

GSC_CLIENT_SECRET = "GOCSPX-xxxxxxxxxx"

Étape 6 — Obtenir le Refresh Token

C'est l'étape finale. Le refresh token permet à Hermes d'appeler l'API GSC sans que l'utilisateur ait à se reconnecter.

1. Ouvre ce lien dans ton navigateur (remplace TON_CLIENT_ID par la vraie valeur) :
https://accounts.google.com/o/oauth2/v2/auth?
  client_id=TON_CLIENT_ID
  &redirect_uri=http://localhost:8502/auth/google/callback
  &response_type=code
  &scope=https://www.googleapis.com/auth/webmasters.readonly
  &access_type=offline
  &prompt=consent
2. Connecte-toi avec le compte Google qui a accès à la Search Console
3. Accepte les permissions
4. Tu es redirigé vers http://localhost:8502/auth/google/callback?code=...
5. Copie le code= dans l'URL (tout ce qui suit code= jusqu'au & ou à la fin)
6. Maintenant, échange ce code contre un refresh token (dans un terminal) :
curl -X POST https://oauth2.googleapis.com/token \
  -d "client_id=TON_CLIENT_ID" \
  -d "client_secret=TON_CLIENT_SECRET" \
  -d "code=LE_CODE_COPIE" \
  -d "redirect_uri=http://localhost:8502/auth/google/callback" \
  -d "grant_type=authorization_code"
{"access_token":"...","refresh_token":"1//...","expires_in":3599}
Le refresh_token (commençant par 1//) est le seul token que tu dois sauvegarder. Il est permanent (ou quasi-permanent). Les access tokens expirent en 1 heure et sont rafraîchis automatiquement.

Étape 7 — Configurer Hermes

Ajoute la dernière clé dans .env :

GSC_CLIENT_ID=123456789-xxxxx.apps.googleusercontent.com
GSC_CLIENT_SECRET=GOCSPX-xxxxxxxxxx
GSC_REFRESH_TOKEN=1//xxxxxxxxxx

Et sur Streamlit Cloud → Secrets :

GSC_CLIENT_ID = "123456789-xxxxx.apps.googleusercontent.com"
GSC_CLIENT_SECRET = "GOCSPX-xxxxxxxxxx"
GSC_REFRESH_TOKEN = "1//xxxxxxxxxx"
Le connecteur est prêt. L'Agent 26 utilise automatiquement ces clés. Si les clés sont absentes, il reste en stub (pas d'erreur, juste pas de données GSC).

Pro tip : Pour éviter d'avoir à refaire l'étape 6 à chaque fois, le refresh token Google ne s'expire que si :
  • Tu le révoques manuellement dans myaccount.google.com/permissions
  • Tu changes ton mot de passe Google
  • L'app passe en statut "testing" et le token a plus de 7 jours
Passe l'app en statut "En production" (même sans vérification) pour que les refresh tokens durent indéfiniment.
