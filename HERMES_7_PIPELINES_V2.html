2. Pipeline Audit de Contenu ~95% · production

Rôle final : Évaluer les pages existantes d'un site sur SEO/AEO/GEO/EEAT/UX et produire un rapport d'audit complet avec recommandations actionnables. Le pipeline ne publie pas — il diagnostique. La réécriture est gérée par le Pipeline 1 (Éditorial).

Validation en production · 23 juin 2026 : Audit complet de hartmann-tresore.fr (PrestaShop e-commerce). CMS détecté à 95%, sitemap PrestaShop parsé (CDATA), 20 URLs auditées en ~30s, scores cohérents (30-34/100 pour les fiches produits, normal pour ce type de page).

Connexion avec le Pipeline 1 : L'audit produit un audit_brief structuré. Le bouton "Réécrire cette page" dans le rapport déclenche le Pipeline Éditorial avec le brief en input.

Interprétation type-aware : Le rapport affiche un bandeau "Comment lire ces scores ?" avec les fourchettes attendues selon le CMS et le type de page (ex: PrestaShop produit = 25-45/100 normal). Chaque page individuelle affiche un indicateur ✓/▲/▼ par rapport à cette fourchette.
