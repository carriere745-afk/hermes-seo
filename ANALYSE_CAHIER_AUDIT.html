1. Verdict global

Le cahier des charges est excellent. Structure complète, agents bien définis, scoring cohérent, connexion propre avec le Pipeline 1. Les retours ChatGPT/Claude/DeepSeek ont été bien intégrés.

Note : 9/10. Les seuls points d'attention sont :

  • 14 agents → 10 suffisent : AC02 (Indexabilité) peut être fusionné avec AC01 (Crawler). AC14 (Connecteur Réécriture) est une fonction de 20 lignes, pas un agent.
  • AC09 (Transparence/Delta DOM) : trop complexe pour une V1. Playwright + comparaison CSS est un chantier à part entière. À reporter en V2.
  • L'estimation de 34h est optimiste. La réalité est plutôt 40-45h en comptant l'intégration et le débogage.

6. Points d'attention

1. AC09 (Transparence / Delta DOM) — TROP COMPLEXE POUR UNE V1
Playwright + comparaison CSS + détection de couleurs = un chantier de 15-20h à lui seul. Le ROI pour une V1 est faible : le cas d'usage "texte blanc sur fond blanc" représente <0.1% des audits. À reporter en V2.
2. AC14 (Connecteur Réécriture) — PAS UN AGENT
C'est un helper de 20 lignes qui prépare l'audit_brief et le passe au Pipeline 1. Pas besoin d'un nœud LangGraph dédié. Une fonction prepare_audit_brief_for_pipeline1() suffit.
3. AC02 (Indexabilité) — FUSIONNER AVEC AC01
Vérifier le status code, le noindex, le canonical — c'est 5 checks triviaux que le crawler peut faire en même temps qu'il fetch la page. Pas besoin d'un agent séparé.
4. Estimation réaliste : 40-45h, pas 34h
Le cahier sous-estime l'intégration, le débogage et les tests. Avec 10 agents (au lieu de 14), l'estimation corrigée est de 32h + 20% de marge = ~38h.
5. BeautifulSoup vs Playwright — ATTENTION
Le cahier propose Playwright en fallback. Playwright pèse 400MB, nécessite un navigateur Chromium, et ne fonctionne pas sur Streamlit Cloud (sandbox). Pour la V1, BeautifulSoup seul suffit. Le rendu JS peut être simulé via requests avec un user-agent Googlebot.
