7. 16 regles de robustesse (toutes implementees)

  • ✅ Contrats Pydantic — tous les agents ont des modeles valides
  • ✅ Superviseur (T00) — consentement obligatoire, bornes, rate limiting
  • ✅ Defensif uniquement — pas de test d'intrusion, pas de scanning
  • ✅ Confidence scoring — High/Medium/Low sur chaque issue
  • ✅ Source de preuve — agent source + URL + observed + rule
  • ✅ do_not_recommend_if — pas de reco absurde (ajouter 300 mots a une CGU)
  • ✅ Adaptatif — seuils par type de page
  • ✅ CMS-aware — recos localisees (WP→Yoast, PrestaShop→Module SEO)
  • ✅ Fallback cascade — GSC→PSI→heuristique
  • ✅ Auto-disable — KE apres 402, PSI apres 429
  • ✅ Deduplication — T15 consolide par URL+categorie
  • ✅ Exports multiples — HTML, JSON, CSV, PDF
  • ✅ Routage inter-pipelines — T18 → P2/P4/P5/P6/P7
  • ✅ Rate limiting + robots.txt
  • ✅ Skip intelligent — T20 (logs) skip auto si pas de donnees
  • ✅ Tracabilite — chaque issue documentee avec donnee observee + regle declenchee

10. Methode de travail

  1. L'utilisateur definit le process (quel pipeline, quels agents, quel ordre)
  2. L'utilisateur ecrit le cahier des charges
  3. Claude implemente agent par agent, dans l'ordre defini
  4. Claude fait les tests et verifications
  5. Claude connecte au pipeline existant sans casser ce qui marche
  6. L'utilisateur valide chaque agent ou groupe d'agents
  7. L'utilisateur teste en conditions reelles
